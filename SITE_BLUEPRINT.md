# サイト設計図 (SITE_BLUEPRINT.md)

このドキュメントは、「京都朝鮮第二初級学校オモニ会」の物品販売注文受付サイトの設計図です。エクセルデータ（オモニ会物販注文書.xlsx）を基に作成しています。

---

## 1. プロジェクト概要

*   **サイトの目的**: オモニ会で販売している食品（キムチ、チャンジャ、うどん、タレ等）のオンライン注文受付。
*   **ターゲット層**: 保護者会の会員、学校関係者、地域の方々など。
*   **デザインの印象**: 食欲をそそる暖色系をベースにした、シズル感のある美味しそうなデザイン。

---

## 2. ページ構成 (シングルページ構成)

1ページ内に以下のセクションを配置したレスポンシブなランディングページ (LP) とします。

1.  **ヒーローセクション (Hero)**:
    *   美味しそうな食品画像と「オモニ会 物品販売」の文字。
    *   注文締切日・お渡し日の重要なスケジュールをわかりやすく提示。
    *   「商品一覧を見る」ボタン。
2.  **スケジュールと受取方法 (Notice)**:
    *   **注文締切日**: 4月28日 (火)
    *   **お渡し日**: 5月16日 (土) 午後より（学校受取は14時～15時の間）
    *   **お支払い方法**: 代金引換
3.  **商品一覧 (Products)**:
    *   エクセルデータを反映した計21品目のカード型リスト。
    *   後日写真を簡単にアップロードできるよう、画像フォルダのファイル名規則を定義します（詳細は後述）。
    *   各商品に数量選択と「カートに追加」ボタン。
4.  **ご注文手続き・カート (Cart / Order Form)**:
    *   カート内の合計金額の自動計算。
    *   注文者情報の入力フォーム:
        *   **注文者名** (必須)
        *   **支部 / 分会** (任意/必須はご指定による)
        *   **電話番号** (必須)
    *   「注文を送信する」ボタン。
5.  **フッター (Footer)**:
    *   保存方法 of 注意（例：「※キムチ、チャンジャは冷蔵保存にて2週間以内にお召し上がりください。」）。
    *   学校へのお問い合わせ先（電話 075-861-0608 / FAX 075-861-0341）。

## 4. 写真を後から簡単に変更する仕組み (アセット構成)

開発時の商品画像は、後で簡単に差し替えられるよう以下のようなファイル構成で作成します。

```text
サイト/
├── index.html          (メインのHTML)
├── style.css           (デザイン定義のCSS)
├── script.js           (カート機能やメール送信のJavaScript)
└── images/
    ├── hero-bg.jpg     (メインビジュアル画像)
    └── products/       (商品画像を格納するフォルダ)
        ├── kimchi_hakusai_500.jpg
        ├── kimchi_hakusai_1000.jpg
        └── (各商品の画像ファイル...上記の表の通り命名)
```

### 写真の更新手順:
1. 後日、食品の写真を撮影します。
2. 写真の名前を上の表に記載されている「仮画像ファイル名」（例: `kimchi_hakusai_500.jpg`）に書き換えます。
3. `images/products/` フォルダの中のファイルを上書きするだけで、**HTMLファイルを1文字も修正することなくサイトの画像が自動で切り替わります**。

*※初期開発時は、プレースホルダーとして美味しいイメージが伝わる「仮画像（ダミー画像）」をこちらで配置しておきます。*

## 5. 技術スタック & 注文管理フロー

*   **言語**: 純粋な HTML5, Vanilla CSS, JavaScript (React等は使用せず、ブラウザだけで動く構成)
*   **レスポンシブWebデザイン**: CSS Media Queriesを活用し、スマホ・タブレット・PC全てに最適化。
*   **メール送信フロー**:
    1.  注文フォームで「注文確定」を押す。
    2.  JavaScriptが動作し、入力内容とカート情報をまとめて整理。
    3.  **EmailJS** を通じて、あらかじめ設定された管理者のメールアドレスに注文内容が自動で送信される。
    4.  送信完了後、画面上に「ご注文ありがとうございました」と案内を表示。
